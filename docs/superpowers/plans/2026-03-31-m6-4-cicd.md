# M6.4 CI (Quality Gates) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implementar CI en GitHub Actions para PR/push a `main` con quality gates obligatorios (`lint`, `typecheck`, `test`, `build`) sin despliegue automatico.

**Architecture:** Se crea un workflow unico en `.github/workflows/ci.yml` con jobs separados por gate para aislar fallos y facilitar branch protection. Cada job usa Node 20, `npm ci`, cache npm y timeout definido. Se agrega evidencia operativa en runbook para validar required checks y corrida exitosa.

**Tech Stack:** GitHub Actions, Node.js 20, npm, Next.js 15, Express, TypeScript, Jest.

### Task 2: Implementar job `lint`

**Files:**

- Modify: `.github/workflows/ci.yml`

- [ ] **Step 1: Write failing check for missing `lint` block**

Run: `rg -n "^\s{2}lint:|name: lint" .github/workflows/ci.yml`
Expected: sin matches para bloque `lint` (pre-condicion roja).

- [ ] **Step 2: Add `lint` job in workflow**

```yaml
jobs:
  lint:
    name: lint
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm --prefix backend ci
      - run: npm run lint
```

- [ ] **Step 3: Run local gate command and capture current result**
